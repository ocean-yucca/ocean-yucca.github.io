CI/CD YAML Syntax
Global Keywords
  • stages
    • The names and order of the pipeline stages
    • Default stages
      • .pre, build, test, deploye, .post
    • Jobs in the same stage run in parallel
    • Jobs in the next stage run after the jobs from the previous stage complete successfully
  • image
    • Defines the default Docker image used by all jobs unless a specific job overrides it
    •
    Job Keywords
  • stage
    • Define which stage a job runs in
    • Default stage is test
  • image
    • Specify the Docker image for a job to run inside
  • tags
    • List of tags that are used to select a runner
  • script
    • Shell script that is executed by a runner
  • needs
    • Defines explicit job dependencies to override stage-based sequencing
    • Start when dependencies finish even it belongs to a later stage
  • dependencies
    • Define from which jobs to fetch artifacts
